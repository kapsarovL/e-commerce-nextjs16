# Security Policy

## Supported Versions

Only the latest release on `main` receives security updates.

| Branch | Supported |
| ------ | --------- |
| `main` | yes       |
| others | no        |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report them privately via [GitHub Security Advisories](../../security/advisories/new).
Include as much detail as possible:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Affected versions and components

You can expect an acknowledgement within **48 hours** and a resolution or status
update within **7 days**. If the vulnerability is accepted, a fix will be released
as soon as possible and credit will be given in the release notes (unless you
prefer to remain anonymous).
